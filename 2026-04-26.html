Pator S. Serialisation Strategy Matters: How FHIR Data Format Affects LLM Medication Reconciliation. arXiv:2604.21076v1. 2026 Apr 22.
https://arxiv.org/abs/2604.21076
Overview: Systematic comparison of 4 FHIR serialisation strategies (Raw JSON, Markdown Table, Clinical Narrative, Chronological Timeline) across 5 LLMs × 200 synthetic patients × 4,000 inference runs on medication reconciliation. Two key methodology findings: (a) optimal serialisation INVERTS with model size — Clinical Narrative beats Raw JSON by up to 19 F1 for small models, but Raw JSON catches up or wins at 70B; (b) omission, not fabrication, is the dominant error mode in medication reconciliation across all formats and model sizes. Task type: Extraction / medication reconciliation. LLM system: 5 LLMs spanning small to 70B. Signal type: Explicit. Setting: Synthetic FHIR-compatible patient data (no real EHR). Volume: 200 patients × 4 formats × 5 models ≈ 4,000 inference runs. Human review: N/A — automated benchmark.
Eval question: When evaluating LLM extraction from structured clinical data (FHIR), how much of the apparent model-quality difference is actually a serialisation-strategy difference — and does the answer change with model scale?
Methodology: Generate 200 synthetic patients with FHIR-compatible medication histories. Render each in 4 serialisation strategies (Raw JSON / Markdown Table / Clinical Narrative / Chronological Timeline). Run reconciliation prompt on 5 LLMs spanning scale. Score precision/recall/F1 vs reference. Decompose error types (omission vs fabrication vs distortion). Cross-tabulate format × model size.
Measurement: Precision/Recall/F1 per format × per model. Format effect size: up to 19 F1 between best and worst serialisation (small-model tier). Format effect inverts at 70B scale. Error decomposition: omission rate >> fabrication rate across all conditions.
Key findings: Serialisation alone moves F1 up to 19 points — comparable to switching model families. The 'best' format depends on model size: human-style Clinical Narrative helps small models; structured Raw JSON wins at 70B. Omission dominates fabrication, consistent with PQ-83 (event-based summarization) and PQ-108 (inference-aware vs literal hallucination) but now in the structured-data extraction setting.
Soft spots: Single-author preprint, not yet peer-reviewed. SYNTHETIC patients, not real EHR. Medication reconciliation only — format effect on other extraction tasks (problem lists, allergies, labs) untested. 5-model selection details limited from abstract. '19 F1' is a max, not an average. The 70B-tier reversal could be a single-model artefact rather than a general scaling pattern.
Strengths: Fully crossed factorial design (4 formats × 5 models × 200 patients). Quantitative anchor (19 F1) for an evaluation axis (serialisation) that is rarely controlled in extraction studies. Model-size-dependent inversion is genuinely novel — challenges assumption that prompt-engineering best practices transfer across scales. Reinforces omission > fabrication finding from PQ-83 / PQ-108 in a different context. Direct relevance to FHIR-native deployment patterns.
Relevance: Layer 1: Extraction (especially structured-data → free-text reconciliation), prompt engineering, model selection. Layer 2: Direct relevance to VAP Extraction kit (does not currently control for input format) and Lung Cancer Screening Eligibility kit (extracts from structured EHR fields). Suggests model-size-aware evaluation pattern: when benchmarking the same kit across model tiers, input serialisation is a confound, not a hyperparameter. Reinforces KTP #9 (input perturbation as adversarial axis, post-PQ-91 / PQ-107) by adding 'input format' as a perturbation dimension.
Verdict reasoning: The model-size-inversion is methodologically consequential — most extraction studies hold input format constant and treat differences as model-quality differences. If real, this is a genuine confounder in cross-model comparisons. However, single-author preprint + synthetic patients + medication-reconciliation-only is enough soft-spot weight to land at 'verify' not 'promote.' Ideal candidate for a 10-field card now and a follow-up in 6 months on peer-review status and replication on real EHR data. MedicalBench: Evaluating LLMs Towards Improved Medical Concept Extraction. medRxiv 2026.04.12.26350704. 2026 Apr 12. (Author list to be confirmed on full-paper read.)
https://www.medrxiv.org/content/10.64898/2026.04.12.26350704v1
Overview: Benchmark for medical concept extraction using 823 MIMIC-IV examples with sentence-level evidence spans and a deliberate 'implicit-positive' / 'confusable-negative' test design. Implicit-positive cases require LLM to infer a clinical concept from indirect evidence (e.g., long-term metformin → diabetes); confusable-negative cases test whether the LLM resists assigning a label when surface text superficially suggests one but evidence does not support it. Sentence-level evidence spans demand transparency: the LLM must point to the source sentence supporting each extraction. Task type: Extraction (medical concept extraction with evidence grounding). Signal type: Mixed — explicit when concept is stated; implicit-positive cases require inference. Setting: MIMIC-IV (US tertiary academic ICU notes). Volume: 823 evaluation examples. Human review: gold-standard annotation underlies the benchmark.
Eval question: Does an LLM correctly extract medical concepts when the evidence is IMPLICIT (must be inferred) or CONFUSABLE (surface text suggests a label that the evidence does not support)? And does it ground each extraction in a specific source sentence?
Methodology: 823 medical concept extraction tasks drawn from MIMIC-IV. Each task has an associated sentence-level evidence span. Categories: explicit positive, implicit positive (inference required), explicit negative, confusable negative (surface superficially supports a label that evidence does not). LLMs evaluated on label correctness AND evidence-span correctness. Per-category performance breakdown.
Measurement: Standard extraction metrics (precision/recall/F1) per category. Evidence-span correctness as a separate axis (does the LLM cite the right sentence?). Disaggregated reporting: explicit vs implicit positive; explicit vs confusable negative.
Key findings: To be confirmed on full-paper read. Expected pattern (based on the test design): LLMs strong on explicit positives, weaker on implicit-positive and confusable-negative cases. Sentence-level evidence-span correctness will likely be lower than label correctness, exposing a transparency gap.
Soft spots: medRxiv preprint, not yet peer-reviewed. MIMIC-IV is a single-institution US tertiary corpus — generalisation to community / non-US / non-English untested. Concept extraction is one extraction subtype — generalisation to medication / procedure / dose extraction needs verification. Author list and full results not yet independently verified by main scout session this cycle (subagent's WebFetch confirmed the medRxiv landing page exists; full-paper details pending). 'Implicit positive' inference is itself a judgment — the gold-standard annotation protocol's reliability is a key validity question.
Strengths: Sentence-level evidence spans force evaluation of transparency, not just label accuracy. Implicit-positive / confusable-negative categories explicitly probe the inference-aware-vs-literal distinction we anchored from PQ-108 — but in the EXTRACTION context, not summarization. Real MIMIC-IV data. Per-category disaggregation prevents aggregate metrics from masking sub-population failures. Directly fills a gap for evaluating extraction where the gold standard requires clinical inference.
Relevance: Layer 1: Extraction, evidence-grounding, inference-aware evaluation, MIMIC-IV-based benchmarking. Layer 2: Reinforces and extends PQ-108 (inference-aware-vs-literal hallucination) into extraction. Relevant to VAP Extraction kit (implicit-positive cases — e.g., 'PEEP 12 with bilateral infiltrates' implies VAP without using the word) and Lung Cancer Screening Eligibility kit (smoking status often requires inference). Adds an evidence-span correctness axis currently missing from Extraction §2.
Verdict reasoning: Implicit-positive / confusable-negative test design is methodologically interesting and directly extends PQ-108's inference-aware framing into the extraction setting. medRxiv preprint status, full paper not yet read end-to-end this cycle, and unverified author list keep this at 'verify' rather than 'promote.'
